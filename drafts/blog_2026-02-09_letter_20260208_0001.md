# Building a Liquid-Glass Marketplace: From Zero to MVP in Next.js 15

Last week, I embarked on building "Das Messer" — a German knife vendor marketplace that needed to feel premium, trustworthy, and distinctly European. What started as a landing page concept evolved into a full-featured multi-tenant platform with a custom design system I'm calling "liquid-glass."

Here's the story of how I went from empty repository to deployed MVP, including the technical hurdles that almost derailed the project.

## The Vision: More Than Just Another Marketplace

The goal was ambitious but clear: create a platform that connects German knife enthusiasts with artisan bladesmiths and premium vendors. But this wasn't going to be another generic marketplace clone.

**Key Requirements:**
- **Liquid-glass aesthetic** — Premium feel with glassmorphism design
- **German-first content** — Native language, local business practices
- **Multi-tenant architecture** — Scalable vendor management
- **SEO-optimized** — Comprehensive knife knowledge base
- **Admin-friendly** — Dashboard for managing vendors, leads, and analytics

## The Tech Stack: Modern but Proven

I chose **Next.js 15** as the foundation, paired with:
- **PostgreSQL** (Neon) for reliable data persistence
- **Prisma** for type-safe database operations
- **NextAuth.js** for authentication with role-based access
- **Tailwind CSS** with custom glassmorphism components
- **TypeScript** throughout for better developer experience

The database schema evolved to support 8 core models: `Tenant`, `User`, `Lead`, `Vendor`, `VendorAd`, `Page`, `SeoKeyword`, and `PageView` — everything needed for a multi-tenant marketplace with analytics.

## Designing the Liquid-Glass Experience

The most unique aspect of this project was the custom design system. I created a glassmorphism theme that feels both modern and premium:

```css
.glass {
  background: rgba(255, 255, 255, 0.1);
  backdrop-filter: blur(10px);
  border: 1px solid rgba(255, 255, 255, 0.2);
}

.glass-card {
  background: rgba(255, 255, 255, 0.05);
  backdrop-filter: blur(15px);
  border: 1px solid rgba(255, 255, 255, 0.1);
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.1);
}
```

The result? UI components that seem to float above richly textured backgrounds, perfect for showcasing premium knife photography.

## Content Strategy: Expertise-Driven SEO

Instead of thin product pages, I built comprehensive knowledge sections:
- **Material Guide** — Steel types, HRC ratings, handle materials
- **Manufacturing Deep-dive** — Forging vs. stock removal, heat treatment
- **Sharpening Mastery** — Tools, techniques, maintenance schedules

Each page includes schema.org structured data and German-optimized meta tags. The FAQ component automatically generates JSON-LD for search engines.

## The Architecture: Public + Admin Separation

Here's where Next.js 15's route groups really shine. I structured the app as:

```
src/app/
├── (public)/           # Public-facing pages
│   ├── layout.tsx      # Header + Footer
│   ├── page.tsx        # Homepage
│   └── produkte/       # Product categories
├── admin/              # Dashboard area
│   ├── layout.tsx      # Admin navigation
│   ├── login/          # Auth gate
│   └── dashboard/      # Analytics & management
└── api/                # Backend endpoints
```

This separation keeps concerns clean while sharing components where it makes sense.

## Lessons Learned: The Pain Points That Taught Me

### 1. Route Group Layout Conflicts

**The Problem:** Initially, I put the Header and Footer in the root layout. This meant admin pages also got the public navigation, creating a confusing user experience.

**The Solution:** Next.js route groups saved the day. By creating a `(public)` group with its own layout, I could keep public and admin areas completely separate while sharing the root layout for fonts and global styles.

### 2. Auth Middleware Redirect Loops

**The Problem:** I tried to handle auth checks in the admin layout using `redirect()`, but this created infinite loops since `/admin/login` is a child route of `/admin`.

**The Solution:** Moving auth logic to `middleware.ts` with pattern matching solved this elegantly:

```typescript
export { default } from "next-auth/middleware"

export const config = {
  matcher: '/admin/((?!login).*)'
}
```

Now the middleware only protects admin routes *except* the login page.

### 3. Prisma Environment Variable Hell

**The Problem:** The Prisma CLI reads `.env` while Next.js reads `.env.local`. Running `prisma db push` failed with "Environment variable not found."

**The Solution:** I maintain both files (both gitignored) with identical database credentials. It's not DRY, but it works reliably across both tools.

## The MVP Feature Set

After two weeks of development, the platform includes:

**Public Features:**
- Responsive homepage with trust indicators
- Three product category pages (kitchen knives, outdoor knives, collections)
- Vendor showcase with live advertisements
- Comprehensive knife knowledge base
- Lead generation forms throughout

**Admin Dashboard:**
- Analytics overview (leads, page views, vendors)
- Vendor and advertisement management
- SEO keyword tracking with performance tips
- Content management for markdown files

**Technical Infrastructure:**
- Multi-tenant database design
- Page view tracking with device detection
- CI/CD pipeline with GitHub Actions
- Vercel deployment configuration

## What's Next: Production Readiness

The MVP is functionally complete, but production deployment requires:

1. **Legal compliance** — Impressum and Datenschutz pages (required in Germany)
2. **Content optimization** — Replace Unsplash placeholders with professional knife photography
3. **Enhanced admin** — CRUD operations for vendors and advertisements
4. **SEO finishing touches** — Sitemap generation and robots.txt
5. **Security hardening** — Change demo passwords and audit authentication flows

This project reminded me why I love full-stack development. There's something satisfying about building a platform that celebrates traditional craftsmanship (German bladesmithing) using modern web technologies.

The liquid-glass design system turned out better than I expected — it strikes the right balance between premium aesthetics and functional usability. The multi-tenant architecture provides room to grow, and the German-first content strategy should give us a strong SEO foundation in our target market.
