## 3. The "Pain" Log (CRITICAL)
* **Plan import paths were WRONG — do not trust the plan's guesses.** Correct (matches `src/app/admin/content/[id]/page.tsx`): `TElement` from `@udecode/plate` (NOT `@udecode/plate-common`); `markdownToPlate`/`plateToMarkdown` from individual files `@/components/admin/editor/serialization/{markdownToPlate,plateToMarkdown}` (NOT a barrel); `PlateEditor` via `dynamic(() => import('...').then(m => ({ default: m.PlateEditor })), { ssr:false })`; params via `useParams()` (NOT `use(params)`).
* **`npm run lint` is UNUSABLE** — `next lint` is unconfigured/interactive and prompts for ESLint setup. **Use `npx tsc --noEmit` + `npm run build`** as the type/compile gate instead.
* **Systemic JSON-LD XSS:** the shared `src/components/JsonLd.tsx` rendered structured data with `dangerouslySetInnerHTML` UNescaped. Fixed with `safeJsonLd()` escaping `<`/`>`/`&` to unicode (commit `dfbb628`, also touched FAQ.tsx + SectionStructuredData.tsx). Note: bio prose is safe because it renders via react-markdown WITHOUT rehype-raw.
* **Slug umlaut bug (fixed):** `normalizeSlug` must transliterate ä/ö/ü/ß BEFORE `.normalize('NFKD')`, else NFKD strips them to bare vowels (`Motörhead→motorhead`). Now correct → `motoerhead`.
* **Vercel bare alias:** `--prod` deploy auto-aliases to a generated `project-*.vercel.app`; had to manually `vercel alias set <deployment> e-ventschau.vercel.app` (same as the greenfield gotcha).
* **Prisma reads `.env` not `.env.local`** (DATABASE_URL); never commit `.env*`.
