## 3. The "Pain" Log (CRITICAL)
* **`vercel --prod` does NOT update the custom alias** `e-ventschau.vercel.app` — it stays pointed at an OLD deployment (which, not knowing `noir`, rendered `eventschau` via graceful getTheme fallback, masking the change). ALWAYS re-point after every prod deploy: `vercel alias set <deployment-url> e-ventschau.vercel.app`, then curl the canonical domain (not the deployment URL) to verify.
* **`git commit` swept in pre-staged WIP:** several of the user's files were already in the index, so `git add <myfile> && git commit` included them under my message. Fix was `git reset --soft HEAD~1` → re-stage only mine → re-commit → re-stage their files. LESSON: check `git status` staged column before committing; stage explicitly.
* **"cloud_name is disabled"** = Cloudinary got an `undefined`/empty cloud_name. Here it meant the 3 `CLOUDINARY_*` env vars were never set on the e-ventschau Vercel project (only DB/auth vars existed). NOT caused by the dasmesser rename (that only changed the Cloudinary *folder*, which is harmless).
* **`npm run lint` is unusable** (interactive next lint). Verify with `npx tsc --noEmit` + `npm run build` only.
* **Shared prod DB:** dev DB == production Neon DB. Never flip `themeSlug` in DB before the code is deployed (old deploy falls back gracefully, but still). Deploy code first, then flip.
* *Note:* Do not retry committing with `git add <one file>` while other files are staged; do not trust that `vercel --prod` updated the canonical domain.
